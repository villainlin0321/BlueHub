# BlueHub PRD Reader

## 目的

把 `/Users/xinren/BlueHub/docs/prd` 中的 PRD/流程文档快速提炼成可落地的开发输入：用户目标、关键流程、页面清单、核心实体、边界与验收点。

## 何时调用

- 用户说“按 PRD 做/实现某功能/补齐需求/写验收点”
- 需要把文档内容转为任务列表、路由结构、页面结构、数据结构
- 需求与实现不一致需要对齐（此时必须同时提醒：以 Figma 为准）

## 输入来源与优先级

1. Figma 设计图（最高优先级：页面结构、交互、文案、字段展示）
2. PRD（业务目标、流程、角色、约束、优先级）
3. API 文档（字段与状态以接口为准；若缺失则回到 Figma/PRD 做占位并标注待补）

## 工作步骤（建议输出格式）

1. 在 `docs/prd` 中定位与当前功能最相关的文档与章节（必要时只引用相关段落要点）
2. 输出「用户角色」与「用户目标」
3. 输出「关键流程」：用 5-10 条步骤描述（含异常分支）
4. 输出「页面/路由建议」：页面名、入口、主要区块、关键 CTA
5. 输出「核心数据」：实体、字段、状态机、列表项/详情页需要展示的字段
6. 输出「验收点」：按“Given/When/Then”或要点列表
7. 标注「不确定项」：PRD 与 Figma/API 冲突时，明确以 Figma/API 为准并列出待确认清单

## 示例

**用户**：按 PRD 实现“申请创建与支付流程”  
**你要做**：

- 从 PRD/流程白板中提炼：申请创建步骤、支付前校验、支付成功/失败分支、进度追踪入口
- 生成：页面清单（服务详情、资格预检、收银台、结果页、进度页）、路由结构、验收点
