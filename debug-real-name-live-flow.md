## 现象

- 用户要求改为正常流程，不接受 mock 数据。

## 待验证链路
1. 求职者登录并进入实名认证页
2. 选择真实本地测试图片
3. 调用真实文件上传链路
4. 调用真实 `/users/me/real-name-verify`
5. 刷新当前用户资料并在我的页体现已实名

## 初始假设
1. 实名流程失败在图片选择阶段，Patrol 难以稳定穿透 iOS 系统相册选择器。
2. 实名流程失败在文件上传阶段，真实本地图片路径或上传预签名链路存在问题。
3. 实名流程失败在实名认证提交阶段，`/users/me/real-name-verify` 返回业务错误。
4. 实名流程实际上提交成功，但 `refreshCurrentUser()` 或页面刷新时序导致 UI 未及时更新。
5. 真实流程可走通，当前缺的只是稳定的自动化输入方式，而不是接口本身有问题。

## 证据计划
- 先定位可用真实测试图片与现有 Patrol/Widget 键位
- 再以非 mock 方式复现一次流程
- 采集运行日志、接口响应与页面截图
